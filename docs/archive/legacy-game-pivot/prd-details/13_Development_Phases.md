## 2. Phase 1: 基础框架 (Foundation)

### 2.1 目标

> 搭建可运行的3D世界，玩家能够创建角色并在场景中自由移动

### 2.2 核心任务

#### 2.2.1 Godot客户端

```
□ 项目初始化
  ├── □ Godot 4.3 项目创建
  ├── □ 目录结构搭建
  ├── □ 基础场景层级设计
  └── □ 资源管理系统

□ 中央广场场景
  ├── □ 地形创建 (100m x 100m)
  ├── □ 基础光照设置 (日/夜循环)
  ├── □ 天空盒配置
  ├── □ AI生成3D模型导入
  │   ├── □ 传送门模型 x3
  │   ├── □ 装饰建筑 x5
  │   └── □ 植被/道具 x10
  └── □ 碰撞体设置

□ 玩家角色系统
  ├── □ CharacterBody3D 控制器
  ├── □ 键盘WASD移动
  ├── □ 鼠标视角控制
  ├── □ 基础动画状态机
  │   ├── □ Idle
  │   ├── □ Walk
  │   ├── □ Run
  │   └── □ Jump
  ├── □ 第三人称相机
  └── □ 简易捏脸系统 (3-5个预设)

□ 基础UI
  ├── □ 主菜单界面
  ├── □ 角色创建界面
  ├── □ 游戏内HUD (HP/等级/金币)
  └── □ ESC暂停菜单
```

#### 2.2.2 Python后端

```
□ 项目搭建
  ├── □ FastAPI项目初始化
  ├── □ Docker开发环境
  ├── □ PostgreSQL + Redis配置
  └── □ 日志系统

□ 用户系统API
  ├── □ 注册/登录
  ├── □ JWT认证
  ├── □ 角色创建
  └── □ 角色数据存储

□ 基础架构
  ├── □ 配置管理
  ├── □ 错误处理
  ├── □ 请求/响应中间件
  └── □ 单元测试框架
```

### 2.3 交付物

| 交付物 | 验收标准 |
|--------|----------|
| 可运行Demo | 本地启动，创建角色，进入中央广场 |
| 角色移动 | WASD移动流畅，动画切换正确 |
| 数据持久化 | 退出后重新登录，角色数据保留 |

### 2.4 里程碑检查点

- [ ] M1.1: Godot项目能运行空场景
- [ ] M1.2: 玩家角色可在场景中移动
- [ ] M1.3: 后端API可注册/登录
- [ ] M1.4: 前后端联调完成

## 3. Phase 2: 核心玩法 (Core Gameplay)

### 3.1 目标

> 实现NPC对话系统，玩家能与AI驱动的NPC进行有意义的对话

### 3.2 核心任务

#### 3.2.1 NPC系统

```
□ NPC基础框架
  ├── □ NPC场景模板
  ├── □ NPC状态机 (Idle/Walking/Talking)
  ├── □ 交互检测 (Area3D)
  └── □ 对话触发机制

□ 核心NPC实现 - 小北 (Bei)
  ├── □ 3D模型导入 + 骨骼绑定
  ├── □ 表情BlendShape
  ├── □ 待机动画序列
  ├── □ 对话动画
  └── □ 简单口型同步

□ 其他NPC占位
  ├── □ 艾拉 (Guide) - 基础功能
  ├── □ 莫莫 (Merchant) - 基础功能
  └── □ 克洛诺斯 (Quest) - 占位
```

#### 3.2.2 AI对话系统

```
□ 后端对话服务
  ├── □ LLM服务抽象层
  ├── □ Qwen API集成
  ├── □ Claude API集成 (情感对话)
  ├── □ 流式响应实现
  └── □ 对话上下文管理

□ 对话记忆系统
  ├── □ 短期记忆 (Redis)
  ├── □ 长期记忆 (PostgreSQL)
  ├── □ Qdrant向量检索集成
  └── □ 记忆摘要生成

□ NPC人设系统
  ├── □ System Prompt管理
  ├── □ 动态情感适配
  └── □ 好感度系统

□ 客户端对话UI
  ├── □ 对话框界面
  ├── □ 文字打字机效果
  ├── □ 快捷回复选项
  ├── □ 对话历史查看
  └── □ 语音输入按钮 (UI占位)
```

#### 3.2.3 探索系统

```
□ 镜像城场景
  ├── □ 城市地形 (200m x 200m)
  ├── □ 建筑群 (AI生成导入)
  ├── □ 赛博朋克光效
  └── □ 环境音效

□ 场景切换
  ├── □ 传送门交互
  ├── □ 加载界面
  └── □ 场景资源管理

□ 物品收集
  ├── □ 可拾取物品
  ├── □ 背包系统
  └── □ 物品使用
```

### 3.3 交付物

| 交付物 | 验收标准 |
|--------|----------|
| NPC对话 | 与小北对话，AI回复流畅、有情感 |
| 情感识别 | 发送悲伤文字，NPC给予安慰回复 |
| 记忆系统 | 提及之前聊过的话题，NPC能想起 |
| 镜像城探索 | 从中央广场传送到镜像城 |

### 3.4 里程碑检查点

- [ ] M2.1: 与占位NPC进行一次完整对话
- [ ] M2.2: Claude API情感对话调通
- [ ] M2.3: 对话历史可查看
- [ ] M2.4: 镜像城场景可进入探索

## 7. 风险管理

### 7.1 技术风险

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| AI服务不稳定 | 高 | 高 | 多服务商备份、降级方案 |
| Godot性能瓶颈 | 中 | 高 | 早期性能测试、LOD策略 |
| 网络同步延迟 | 中 | 中 | 客户端预测、区域服务器 |
| 3D模型质量 | 中 | 中 | 多工具尝试、人工优化 |

### 7.2 业务风险

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| AI成本超预期 | 高 | 高 | 免费用户配额限制、混合模型 |
| 用户留存低 | 中 | 高 | 持续内容更新、社交绑定 |
| 竞品冲击 | 低 | 中 | 差异化定位、快速迭代 |
