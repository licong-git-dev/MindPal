## 二、从代码库导出材料的具体方法

### B2 界面截图

**不启动服务也能出截图的做法**：
1. 把 `frontend/*.html` 本地浏览器直接打开（file:// 协议）

### C5 心理危机干预机制说明（2 页附件）

```markdown
# 心理危机干预机制 — MindPal

## 1. 触发条件

5 级危机分类（NONE / LOW / MEDIUM / HIGH / CRITICAL）：

- 108 个关键词（含"想死"/"自杀"/"活不下去"等明确表达）
- 7 个正则模式（含"不想活"/"结束一切"等变体）
- 上下文分析（最近 5 条消息综合判断）

## 2. 响应动作

MEDIUM 及以上级别触发:
  a) LLM 温度从 0.8 → 0.5（更保守的回复）
  b) System prompt 注入危机安全指导
  c) 前端强制展示 crisis banner，包含：
     - 全国心理援助热线: 400-161-9995
     - 北京心理危机研究与干预中心: 010-82951332
     - 生命热线: 400-821-1215
     - 青少年心理援助热线: 12355
  d) 事件写入 crisis_events 表供审计

## 3. 安全响应原则（注入到 LLM prompt）

- 不评判用户感受
- 表达关心，保持冷静
- 绝不说"别想太多"/"想开点"
- 绝不讨论具体自伤方式
- 高/极危级别必须主动提供热线

## 4. 事件统计

近期数据（若有上线记录）:
[截图 crisis_events 表 SQL 查询结果]
```

## 四、常见退回原因及规避

网信办最常见的**退回补正**原因：

| 原因 | 规避方法 |
|-----|---------|
| 界面截图不清晰 / 没有触发关键场景 | 用高分辨率截图，确保 crisis banner、moderation toast 都有 |
| 评测集题目太少或没有代表性 | 每类 100 题起步，真的按监管定的类别出题 |
| 没有覆盖未成年人保护 | 即便产品不面向未成年，也要写"如何识别未成年用户并切换模式" |
| 隐私政策过于简略 | ≥ 3000 字，包含 PIPL 所有法定告知项 |
| 安全负责人没有联系电话 | 必须有真人姓名 + 手机 + 邮箱 |
