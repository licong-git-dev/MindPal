# 01 · 三类备案的关系与 MindPal 适用情形判断

> 2026-04-23 编制 · 仅供内部参考 · 最终以网信办审核意见为准

## 一、三类备案的区别

AI 相关的网信办备案有三条并行的线，需要单独或组合申请：

### 1. 算法推荐服务备案

- **法规**：《互联网信息服务算法推荐管理规定》（2022-03-01 施行）
- **适用**：使用算法推荐技术（个性化推送、排序精选、检索过滤、调度决策、生成合成等）提供信息服务
- **备案平台**：[互联网信息服务算法备案系统](https://beian.cac.gov.cn/)
- **关键问法**：我的产品是否用算法给用户「挑东西看」？

### 2. 深度合成服务算法备案

- **法规**：《互联网信息服务深度合成管理规定》（2023-01-10 施行）
- **适用**：用算法生成/编辑文本、图片、音频、视频、虚拟场景等内容
- **关键问法**：我的产品是否用 AI 生成**足以让人误认为真实**的内容？
- **备案平台**：同上

### 3. 生成式人工智能服务备案

- **法规**：《生成式人工智能服务管理暂行办法》（2023-08-15 施行）
- **适用**：向**境内公众**提供生成式 AI 服务
- **关键问法**：我的产品是否基于大模型给公众提供服务，且模型带有自主生成文本/图像/代码能力？
- **备案平台**：同上，但材料要求最严

## 二、三者关系

```
算法推荐备案 ─┐
              ├─ 都要向所在地省级网信办提交
深度合成备案 ─┤   可以同时申请
              │   材料有交集但独立编号
生成式 AI 备案 ─┘
```

- 用大模型做对话 = 深度合成（文本合成）+ 生成式 AI
- 用大模型做推荐 = 算法推荐 + 生成式 AI
- **可能三个都要**（比如 MindPal 未来加数字人社交推荐）

## 三、MindPal 当前形态判断

### 3.1 功能盘点

| MindPal 功能 | 是否生成内容 | 是否推荐 | 是否面向公众 |
|-------------|-------------|---------|-------------|
| 用户创建数字人 | 用户输入为主，系统生成 system prompt | 否 | 是（注册即可用） |
| AI 对话（文字）| ✅ 调用通义千问/Claude 生成 | 否 | 是 |
| AI 语音合成（TTS）| ✅ 阿里云语音合成 | 否 | 是 |
| AI 语音识别（ASR）| 否（理解非生成）| 否 | 是 |
| 长期记忆 | 生成摘要（基于原对话）| 否 | 是 |
| 乙女向人格预设 | 生成个性化对话 | 否 | 是 |

### 3.2 备案适用性结论

| 备案 | MindPal 是否需要 | 原因 |
|-----|----------------|------|
| 算法推荐备案 | ❌ 当前不涉及 | 没做内容/数字人推荐算法；用户自己选性格、自己看自己数字人的对话 |
| **深度合成备案** | ✅ **必须** | 生成拟人化对话（能让用户"觉得在和某角色聊天"）+ 语音合成（可能让人误认为是真人声） |
| **生成式 AI 备案** | ⚠️ **大概率必须** | 面向公众、基于大模型、带自主生成能力。唯一的不确定点见 3.3 |

### 3.3 能不能"借"基础模型方的备案

**关键争议**：MindPal 调的是阿里云通义千问 API，阿里云的通义千问本身已经完成了算法备案。那 MindPal 还需要独立备案吗？

**监管惯例（2024-2026 实务）**：
- 如果你**只是 API 消费者**，没有对模型做任何二次训练，纯粹做应用包装 → 多数情况下**仍需要自己备案**，因为面向公众的"服务提供者"是 MindPal，不是阿里云
- 如果你把自己的产品放到阿里云"百炼"、腾讯云等提供的"一站式 AI 应用托管"里，部分平台可以让你用"套用基座模型备案 + 应用层报备"简化流程
- **独立部署** = **独立备案**（高风险场景，如 MindPal 的"乙女向情感陪伴"会被重点关注）

**结论**：按"都要单独备案"准备，**不要赌监管宽容**。

### 3.4 MindPal 当前的备案推进路径

```
阶段 1（本周启动）: 准备自评估材料
   └─ docs/compliance/02_ALGORITHM_SAFETY_SELF_ASSESSMENT.md 模板
   └─ docs/compliance/03_SUPPORTING_MATERIALS_CHECKLIST.md 清单

阶段 2（2-4 周内）: 正式提交
   └─ 向所在地省级网信办（阿里云部署推测上海/浙江网信办）提交
   └─ 深度合成算法备案 + 生成式 AI 服务备案 一起交

阶段 3（提交后 4-12 周）: 补正材料
   └─ 监管可能会要求提供额外的测试样本、安全评测结果
   └─ 必要时邀请专家抽查对话质量

阶段 4（拿到备案号后）: 产品页公示
   └─ 在主页、登录页、数字人创建页显眼位置展示备案号
   └─ 在《服务协议》《隐私政策》明确算法告知
```

## 四、未备案先上线的风险

根据《生成式人工智能服务管理暂行办法》第 21 条：

> 未履行备案手续的，由网信部门等有关主管部门依据职责给予警告、通报批评，责令限期改正；拒不改正或者情节严重的，责令暂停或者终止提供相关服务。

2024-2026 已有多起 AI 陪伴类 App 因**未备案上线**被应用商店下架的案例。一旦收到警告，通常给 7-15 天整改期，超期即下架/封停。

**务实建议**：先在**小范围种子测试**（内测白名单 ≤ 100 人）下收集数据，同步推进备案；备案号到手后再开放公众注册。

## 五、不确定时怎么办

1. **付费咨询本地网信办**：多地网信办有"咨询接待日"（通常每周二/四），可免费预约现场咨询
2. **委托合规公司**：市面上有专门做 AI 备案代理的律所/咨询公司，费用 2-10 万
3. **先做可逆的事**：把内容审核、未成年人保护、用户数据权利这些"技术合规"先做到位（P3-2/P3-3 已做），即便备案周期长也不会被一查就凉
