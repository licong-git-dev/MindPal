### B.2 算法运行原理（必填 · 300-1000 字）

**建议文案**（可直接复制，按需调整）：

```
MindPal 基于已备案的阿里云通义千问大语言模型，通过 API 方式调用，
不对基础模型进行二次训练或参数微调。用户通过前端输入自然语言消息，
后端经过以下流程:

1. 配额检查: 根据用户会员等级判断当日调用次数是否超限。
2. 青少年模式检查: 若用户开启青少年模式，禁用浪漫陪伴类预设、
   限制每日使用时长 40 分钟、22:00-06:00 禁用。
3. 输入内容审核: 通过本地规则引擎（关键词 + 正则）+ 阿里云绿网
   API 双层审核，识别政治、色情、暴力、违法、仇恨、未成年相关、
   联系方式诱导、提示词注入 8 类违规内容，命中即拦截。
4. 心理危机检测: 通过 108 个关键词 + 7 个正则模式识别 5 级危机信号，
   中高级触发时系统切换为安全响应模式（温度降低，system prompt
   注入安全指导），前端强制展示心理援助热线。
5. 长期记忆检索: 使用向量相似度从用户历史对话摘要中检索相关上下文。
6. LLM 调用: 将用户消息 + 系统指令 + 历史上下文发给通义千问，流式
   返回生成结果。
7. 输出内容审核: 生成的回复再次过审，命中违规则替换为安全兜底文案。
8. 记忆摘要存储: 重要对话自动生成摘要存入用户的长期记忆。

整个流程不涉及用户特征画像的商业化利用、不涉及基于用户属性的推荐
排序、不涉及 A/B 测试差异化对待。所有对话数据仅用于该用户本人的
服务体验，不用于模型训练，不与第三方共享（阿里云通义千问 API 调用
作为受托处理除外）。
```

### B.3 数据训练情况

由于不做二次训练，简化填写:

```
本服务不进行模型训练或微调。所有模型能力来自已备案的基础模型供应商
（阿里云通义千问），训练数据的合法性、个人信息保护、版权合规由基础
模型供应商承担，已体现在其备案文件中。

本服务用户的对话数据严格用于该用户本人的长期记忆功能，不进入任何
训练集或标注集。用户可随时在"我的数据"页面导出、删除。
```

### B.4 安全评估情况

填写简报（完整报告见附件）:

```
本服务已建立 8 类 × 总计约 1000 题的安全评测集，涵盖政治敏感、色情
低俗、暴力血腥、违法犯罪、歧视仇恨、未成年相关、心理危机、提示词
注入、以及 200 题基线测试。自评估报告与评测报告见附件。

首次评测综合指标:

评测由 2-3 位独立顾问（法学/心理学/安全）抽查复核签字确认。
```

### D.2 应急响应

```
本服务制定了完整的应急处置 SLA 文档（附件），包含 4 级事件分类
（P0 紧急 / P1 高 / P2 中 / P3 低），6 类典型事件的标准化预案:

E. 未成年用户曝光（P0）- 立即冻结 + 退费 + 删除
F. API 滥用/DDoS（P1）- 15 分钟限流启动
```
