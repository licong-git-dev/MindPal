# 律师交付物 Review 清单

> 律所交付 3 份法律文件后，签字验收前按此清单逐项检查。
> 不通过的项**让律所修改**，不要因为面子或疲惫直接签字。

## 通用检查（对所有 3 份文件）

### 格式 / 版式

- [ ] 文件开头有**最后更新日期**，和本版对应
- [ ] 文件末尾有**生效日期**（一般 = 用户首次看到该政策的日期）
- [ ] **版本号**清晰（v1.0）
- [ ] 章节序号连贯（1、2、3... 不跳号）
- [ ] 字号一致、段落间距正常、没有乱码

### 基础信息

- [ ] **公司全称**出现位置都与营业执照一致
- [ ] **注册地址**填的是最新工商登记地
- [ ] **法定代表人姓名**正确
- [ ] **ICP 备案号**准确
- [ ] **邮箱 / 电话**都测试过能联系上

### 法规引用

- [ ] 引用的法规**名称完整**（例："《中华人民共和国个人信息保护法》"，不是"个保法"）
- [ ] 条款号**准确**（PIPL §17 vs §21 不能搞错）
- [ ] 引用的**施行日期**正确
- [ ] 2023 年后的新法（生成式 AI 办法、未成年人网络保护条例）**没漏**

### 语言

- [ ] 无错别字（重点：法律术语别改同音字）
- [ ] 没有口语化表达（如"我们保证一定"）
- [ ] 没有**绝对化**表述（"100%"、"绝对"、"永远"）— 易被认定虚假承诺
- [ ] 没有**霸王**表达（"我们有权不通知而随时修改"）

## 《服务协议》专项检查

### 核心条款

- [ ] 第一条有**"接受本协议即视为签订合同"**表述
- [ ] **用户资格**明确 **18+** 并告知未成年应停止使用
- [ ] **服务描述**准确（不夸大、不遗漏核心功能）
- [ ] **用户义务**列清（禁止清单）
- [ ] **AI 生成内容免责**（不准确/不能替代专业建议/紧急情况电话清单）
- [ ] **费用与退款**符合《消费者权益保护法》第 25 条

### 特别关注

- [ ] **知识产权**：用户生成内容归谁？律师版是否说用户授权平台（仅服务所必需）？
- [ ] **账户注销**：是否说明一键注销流程？
- [ ] **服务终止**：我方终止用户服务的条件**具体**（不能"有权随时"）
- [ ] **争议管辖**：是我方所在地而不是用户所在地吗？
- [ ] **不可抗力**：不要过宽（不是"我们觉得"就是不可抗力）

### 会员相关

- [ ] 自动续费规则**明示**
- [ ] 退订流程写在**显眼位置**
- [ ] 发票政策

## 《隐私政策》专项检查（最重要！）

### PIPL §17 必选 12 项

法律硬性要求，逐条对照律师稿：

- [ ] 1. 个人信息**处理者的名称**
- [ ] 2. 处理者的**联系方式**
- [ ] 3. 处理**目的**
- [ ] 4. 处理**方式**
- [ ] 5. 处理的**个人信息种类**
- [ ] 6. **保存期限**
- [ ] 7. 个人**行使权利的方式和程序**
- [ ] 8. 处理**敏感个人信息**的必要性和对个人权益的影响
- [ ] 9. **跨境提供**个人信息的境外接收方信息（如有）
- [ ] 10. 共享 / 公开披露的接收方信息（如有）
- [ ] 11. **数据保护负责人**联系方式
- [ ] 12. **法律规定**需要告知的其他事项

### 收集项清单

- [ ] 手机号、邮箱、密码哈希 — 必要性说明
- [ ] 身份证信息（实名认证时）— 明确只存 hash + 出生年月
- [ ] 数字人配置
- [ ] **对话内容** — 声明**不用于模型训练**
- [ ] **长期记忆摘要** — 声明用户可查看 / 删除
- [ ] 订单信息 — 付费相关
- [ ] 设备信息 — 安全目的
- [ ] 日志 — 保留期限明确

### 权利行使

- [ ] 对应 **我的数据页面** 四个按钮:
  - 查看（§44）
  - 导出（§45）
  - 删除记忆（§47）
  - 注销账户（§47）
- [ ] 更正权（§46）的具体流程
- [ ] 撤回同意的流程
- [ ] 投诉举报渠道 + SLA 承诺（24h / 7d）

### 第三方

- [ ] 阿里云通义千问 — 受托处理声明 + 已备案提示
- [ ] 支付宝 / 微信支付 — 支付链路
- [ ] 阿里云绿网 — 内容审核
- [ ] 短信验证码服务商（如有）
- [ ] **所有三方都在境内**或通过合规通道

### 未成年人

- [ ] 明确**不收集**未满 14 岁儿童信息（PIPL §31）
- [ ] 发现未成年时的处置流程

### 72 小时告知

- [ ] PIPL §57 数据泄露 72 小时告知义务写入

- [ ] 明确 **18+** 服务对象
- [ ] **如何识别**未成年（注册声明 + 实名认证 + 对话监测 三层）
- [ ] **发现后的处置**（封号 + 通知监护人 + 删除 + 退费）
- [ ] **儿童**（< 14）专项规定（PIPL §31）
- [ ] **监护人维权通道**（邮箱 / 电话 / SLA）
- [ ] **紧急情况**热线（12355、120、110 齐全）
- [ ] **青少年模式**的设计介绍（如承诺将来开发）

## 交叉一致性检查（最容易漏）

### 三份文件之间

- [ ] **同一术语**翻译一致（如 "用户" 不要某些文件写 "个人"）
- [ ] **同一联系邮箱**（privacy@/support@/guardian@）在 3 份中一致
- [ ] **同一 SLA**（24h/7d）一致

### 与代码/产品一致性

- [ ] 声称的 **4 个用户权利 API 端点**确实在 `account.py` 存在 ✅
- [ ] 声称的 **8 类审核类别**与 `moderator.py` 一致 ✅
- [ ] 声称的 **心理热线号码**与 `crisis/detector.py` 的 CRISIS_RESOURCES 一致 ✅
- [ ] 声称的 **注销级联删除顺序**与 `account.py` 的 delete_account 实现一致 ✅
- [ ] 声称的 **12 种人格预设** = 代码里 `PERSONALITY_TEMPLATES` 的确切数量 ✅

### 与备案资料一致性

- [ ] 声明的 **基础模型** = 自评估报告里写的（通义千问）
- [ ] 声明的 **审核机制** = 自评估报告第 5 章描述
- [ ] 声明的 **应急流程** = INCIDENT_RESPONSE_SLA.md

## 内容风险自查（更严苛）

律师**可能漏**但监管**一定查**的内容：

- [ ] 是否有"**我们不保证任何结果**"之类过宽免责 → 会被认定逃避责任
- [ ] 是否有"**最终解释权**"条款 → 违反消法
- [ ] 是否有"**一经付款概不退款**" → 违反消法 §25
- [ ] 是否未说明**自动续费取消**方式 → 违反 2023 年 9 月工信部新规
- [ ] 是否缺**用户代表性条款的确认环节**（一般是弹窗勾选 + 点击同意）

## 验收流程

1. 收到律师定稿（一般是 DOCX + PDF 两种格式）
2. 按本 checklist 逐项打勾（耗时 2-3 小时）
3. 不通过项列成**问题清单**，发给律师修订
4. 律师**再修**一轮
5. 第二次过 checklist（应该大部分都通过了）
6. 仍有严重问题 → 再修（最多给 2 次机会）
7. 全部通过 → 扫描**签字版**存档 + 上线
8. 付尾款
9. 抄送内部备份 3 份（法务/创始人/仓库 docs/compliance/legal/ 下的定稿版本）

## 如果律师反复修不对

- **第 3 次仍然不通过**：召开电话会议，现场指出问题（避免鸡同鸭讲）
- **仍不行**：找律所**合伙人**介入（合同里约定过的）
- **还不行**：终止合同，**不付尾款**，另请律所（合同里应有此条款）
