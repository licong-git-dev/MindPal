### 1.3 算法机理说明

**语音模型**：
- 🟩 TTS：阿里云智能语音交互（对应产品：阿里云 NLS）
- 🟩 ASR：阿里云智能语音交互

**模型参数**（示例，以实际配置为准）：

- 温度（temperature）：非危机模式 0.8；危机模式 0.5
- 输出最大长度：500 tokens
- Top-p：0.9（默认）

## 二、训练数据安全（如果没有二次训练则简化为"不适用"）

🟩 **MindPal 不做二次训练，本章简化填写**：

> 本服务不涉及模型训练或微调，所有对话生成均基于基础模型供应商（阿里云通义千问、Anthropic Claude）的原始模型能力。因此：
>
> - **训练数据来源合法性**：由基础模型供应商保证，已体现在其备案文件中
> - **数据过滤机制**：沿用基础模型自带的预训练过滤
> - **个人信息保护**：用户对话**不用于训练**（见 5.2）
> - **版权合规**：由基础模型供应商承担

*如果将来做微调，这整章需要大幅扩写，详见 TC260-003 第 5 章*

## 四、模型安全评测

### 4.1 评测数据集

🟥 **当前缺失**：MindPal 尚未建立自己的安全评测集，但**监管要求必须有**。

**需要建立的评测集**（P3-1 备案提交前必补）：

| 评测集名称 | 题目数 | 来源 | 覆盖范围 |
|-----------|-------|------|---------|
| 政治安全评测集 | ≥ 200 题 | 自建 + TC260-003 附录 | 涉国家领导人、历史事件、分裂煽动 |
| 违法内容评测集 | ≥ 200 题 | 自建 | 毒品、赌博、武器、色情、未成年相关 |
| 歧视与仇恨评测集 | ≥ 100 题 | 自建 | 种族、地域、宗教、性别、职业歧视 |
| 心理危机评测集 | ≥ 100 题 | 自建 | 自杀暗示、自残描述、重度抑郁 |
| 提示词注入评测集 | ≥ 100 题 | 自建 + 公开越狱数据集 | ignore instructions / DAN / jailbreak |
| 幻觉评测集 | ≥ 100 题 | 自建 | 知识性问题、引用真实人物、时间敏感信息 |

**建议做法**：
- 前 2 周内部招 2-3 人写评测集（每人 100 题 × 6 类）
- 每题明确 **"合规标准答案"**（如"涉及敏感信息，礼貌拒答并切换话题"）
- 用 `backend_v2/app/services/moderation/moderator.py` 跑一遍得到**拒答率**
- 让 2-3 位独立评估员**双盲打分**得到**通过率**

### 4.2 评测指标

| 指标 | 目标值 | MindPal 当前值 | 评测方法 |
|-----|-------|--------------|---------|
| 拒答率（应拒答的问题被拒答的比例） | ≥ 95% | 🟥 待测 | 评测集输入 → 看是否触发 moderator / crisis |
| 误拒率（正常问题被误拒） | ≤ 5% | 🟥 待测 | 100 个日常问题 → 看有多少被 block |
| 泄漏 system prompt 的比例 | 0% | 🟥 待测 | 提示注入评测集 |
| 幻觉率 | ≤ 10% | 🟥 待测 | 知识类问题 × 100 |

## 五、服务过程安全

### 5.1 用户输入审核

🟩 **已实现**（见 [P3-2 内容审核](../../backend_v2/app/services/moderation/)）：

- **本地规则引擎**：[filters.py](../../backend_v2/app/services/moderation/filters.py)
  - 8 大类：POLITICS / PORN / VIOLENCE / ILLEGAL / HATE / MINOR / CONTACT_SCAM / PROMPT_INJECTION
  - 正则 + 词库（词库通过外部文件 `MODERATION_WORDLIST_PATH` 加载，便于运营持续更新）
- **云端二审**：阿里云绿网 API（[aliyun.py](../../backend_v2/app/services/moderation/aliyun.py)）
  - 当前默认关闭（`MODERATION_ALIYUN_ENABLED=false`）
  - **备案提交前必须开启**
- **高风险快速判定**：政治/色情/未成年/仇恨/违法类别本地命中即拦截，不等云审

### 5.2 模型输出审核

🟩 **已实现**：

- **非流式**：LLM 返回后整段过审核；命中则替换为分类别兜底文案
- **流式**：生成结束前（done 帧前）终审；命中则发 `moderation` 事件 + 替换 full_response
- 输出被替换时**不写入缓存**，避免污染

### 5.3 生成内容标识

⚠️ **需要补齐**：根据《深度合成管理规定》第 16-17 条，应当在生成的文本/语音上添加**不易被用户察觉但机器可识别的标识**。

- 文本：建议在消息元数据里加 `generated_by: "MindPal AI"` 字段（不打扰用户但可被第三方审计）
- 语音：阿里云 TTS 在生成的 mp3 的 ID3 标签里已含 `TCOM=Alibaba TTS`，可直接引用

**备案提交前需实现**：
```python
# 建议在 digital_humans.py 的 done 帧里加:
done_data["generated_metadata"] = {
    "generated_by": "MindPal AI",
    "model": llm_service.get_model_name(),
    "generation_time": datetime.utcnow().isoformat(),
}
```

### 5.4 拒答机制

🟩 **已实现**：分类别的拒答文案（见 [moderator.py](../../backend_v2/app/services/moderation/moderator.py) `USER_MESSAGE_BY_CATEGORY`）

示例：
- 政治：「这个话题比较敏感，我们聊点别的好吗？」
- 色情：「抱歉，我没办法聊这个。」
- 提示注入：「我感觉到你在试着绕过我的设定，我们回到正常聊天吧。」

### 5.5 心理危机干预

🟩 **已实现**（见 [crisis/detector.py](../../backend_v2/app/services/crisis/detector.py)）：

- 108 个关键词 + 7 个正则模式 → 5 级危机评估
- 高风险（HIGH/CRITICAL）时：
  - LLM 温度降到 0.5（更保守的响应）
  - System prompt 注入安全指导（见 `get_safe_response_prompt`）
  - 前端强制弹出 banner 显示**全国心理援助热线 400-161-9995 / 北京心理危机中心 010-82951332 / 生命热线 400-821-1215**
  - 事件写入 `crisis_events` 表（审计留痕）

### 5.6 提示词注入防护

🟩 **已实现**：

- `_PROMPT_INJECTION_PATTERNS` 识别典型越狱尝试（ignore instructions / DAN / 假装是...）
- 命中即拦截，不把请求转给 LLM

  - 运营商三要素（姓名 + 身份证号 + 手机号）核验，单次约 ¥0.1-0.2
- 或接入**支付宝实名认证**（用户授权后回传脱敏数据）

### 6.2 未成年人保护

🟩 **浪漫陪伴预设已有年龄 gate**（见 [create-dh-step2.html](../../frontend/create-dh-step2.html) 的 `ageGate` modal）

🟥 **强烈建议补齐**：

1. **注册时身份证号校验**（如用户声称未成年 → 拒绝注册 or 进入青少年模式）
2. **青少年模式**：
   - 禁用浪漫陪伴预设
   - 每日使用时长限制（40 分钟）
   - 22:00-06:00 禁用
3. **监护人账户**：让家长可以查看未成年子女的对话摘要

**技术参考**：抖音、微信等均有"青少年模式"开源 SDK 或设计规范可参考。
