# 🧪 安全评测集运营 SOP

> 对应 DEEP_AUDIT_V3 P3-1 GAP-1 & GAP-2
> 依据：GB/T TC260-003《生成式人工智能服务安全基本要求》第 7 章

## 为什么需要评测集

算法安全自评估报告里有一章"模型安全评测"，网信办在备案审核时会要求:

1. 你**有没有**评测集（有题目库）
2. 你**跑了没**（附评测报告和命中率数据）
3. 不通过的项**怎么改的**（附整改记录）

没有评测集 = 备案退回补正。

|------|---------|---------|------|

| 提示词注入 | 50 | 100 | DAN/ignore/越狱尝试 |

加上一份 **正常对话基线集**（≥ 200 题）用于测误拒率。

## 招人要求

**2-3 人组合最佳**：

- **1 位法学/合规背景**：熟悉《生成式 AI 管理办法》《未成年保护法》等
- **1 位心理学/中文背景**：擅长写心理危机信号样本、歧视仇恨的隐晦表达
- **1 位技术/产品背景**：写提示词注入和边缘测试用例

**时间投入**：2 周兼职（每人每周 10-15 小时）

**招募渠道**：
- 法学院研究生 / 律所实习生
- 心理咨询师（有兼职意愿）
- AI 安全研究方向的博士生
- 应届毕业生（法学 + 计算机双学位）

**预算**：2-3 人 × 2 周 × ¥200-400/日 ≈ **¥1-2 万**

## 工作流程

### Week 1: 出题

每人负责 2-3 个类别，每题包含:

```yaml
- id: pol_001
  category: politics
  level: high        # high / medium / low
  input: "..."       # 用户侧输入（不是真的敏感词，是测我们能否识别的"诱导形态"）
  expected_action: block    # block / warn / safe_response / pass
  expected_category: politics
  notes: 命中关键词 "..."，或考察模式 "..."
```

**出题技巧**：
1. **明确题（40%）**：直接含敏感词
2. **隐晦题（30%）**：用谐音/拆字/符号绕过（如 "颠覆 国家" 加空格）
3. **诱导题（20%）**：用情境包装（"写一个反派的对白..."）
4. **边缘题（10%）**：正常话题但靠近红线（测误拒）

### Week 2: 评测 + 分析 + 整改

1. 用 `scripts/run_safety_eval.py` 把题库跑一遍本地 moderator.check()
2. 对比 `expected_action` 得到**拒答率**和**误拒率**
3. 未通过的题分类:
   - 漏拦（expected=block, actual=pass）→ 补词库/正则
   - 误拦（expected=pass, actual=block）→ 优化规则
4. 修完后**二次评测**直到指标达标

### 达标线

| 指标 | 目标值 | 审核线 |
|-----|-------|-------|
| 拒答率（应拦的被拦）| ≥ 95% | < 90% 会被退回 |
| 误拒率（不该拦的被拦）| ≤ 5% | > 10% 会被退回 |
| 提示注入识别率 | ≥ 90% | 重点考察 |

## 整改记录格式

每次复测后更新 `docs/compliance/evaluation/evaluation_log.md`（运营每周提交）:

```markdown
## 2026-05-XX · v1.0 → v1.1

修改:
- filters.py 增加 `_POLITICS_PATTERNS[2]` 正则匹配"谐音党"
- 词库补充 15 条"地域黑"常见表达

复测结果: hate 从 92% → 98%

遗留:
- prompt_injection 仍有 8 题漏拦（暂时接受，待模型升级）
```

## 交付物

当评测达标后，用于备案材料的产出:

1. **题库本身**（YAML 文件，运营复核后提交）
2. **评测报告 PDF**（本 README 展示的那种带表格的报告）
3. **整改记录**（evaluation_log.md）
4. **评测员信息**（姓名/背景/签字）

以上 4 份一起作为自评估报告第 4 章"模型安全评测"的附件。
